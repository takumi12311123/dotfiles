---
description: Run linter, formatter, tests, then perform code review
---

100行以上の差分がある場合に使用する、包括的なコードチェック&レビューコマンド。

## 実行フロー

1. **コードフォーマット**
   - `make format` を実行
   - フォーマットの問題があれば自動修正

2. **リンター**
   - `make lint` を実行
   - 静的解析でコード品質をチェック
   - エラーがあれば報告し、可能なら修正

3. **テスト実行**
   - `make test-all` を実行
   - 全テストをレース検出ありで並列実行
   - テスト失敗があれば詳細を報告

4. **コードレビュー**
   - 上記すべてがパスした場合のみ実行
   - Reviewエージェント（`review`）を起動
   - 包括的なコードレビューを実施

5. **レビュー結果に基づく修正**
   - Critical/Major issuesがあれば修正
   - 修正後、再度ステップ1-4を実行

## 使用方法

```
/review-with-checks
```

## 注意事項

- すべてのチェックが通らないとレビューは開始されません
- テスト失敗時は原因を特定し、修正してから再実行してください
- レビュー後の修正が完了したら `/explain-implementation` で実装内容を説明できます
